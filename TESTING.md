# Тестирование исправлений планировщика LTE модемов

## Описание исправлений

Исправлена работа запланированной проверки LTE модемов в 11:00 ежедневно.

## Основные изменения

### 1. Упрощена логика планировщика
- Убраны дублирующие задачи и тестовые планировщики
- Оставлена только одна основная задача `lte_daily_check`
- Добавлен мониторинг здоровья планировщика

### 2. Улучшена обработка таймзон
- Добавлена функция `get_next_11am_time()` для корректного вычисления времени
- Автоматическое определение следующего запуска в 11:00
- Корректная работа при смене дат

### 3. Добавлен мониторинг здоровья
- Каждые 6 часов проверяется состояние планировщика
- Автоматическое восстановление утерянных задач
- Принудительный запуск проверки, если она не была выполнена

### 4. Новые команды управления
- `/scheduler` - показать статус планировщика
- `/reschedule` - перезапустить планировщик
- `/status` - показать статус роутеров
- `/test` - тестовая команда

## Тестирование

### 1. Проверка синтаксиса
```bash
python3 -m py_compile bot.py
```

### 2. Тест функции вычисления времени
```bash
python3 -c "
from datetime import datetime, time, timedelta
from zoneinfo import ZoneInfo

def get_next_11am_time(tz):
    now = datetime.now(tz)
    today_11am = datetime.combine(now.date(), time(11, 0, tzinfo=tz))
    
    if now >= today_11am:
        next_run = today_11am + timedelta(days=1)
    else:
        next_run = today_11am
    
    return next_run

tz = ZoneInfo('Europe/Moscow')
next_run = get_next_11am_time(tz)
now = datetime.now(tz)

print(f'Текущее время: {now.strftime(\"%H:%M:%S\")}')
print(f'Следующий запуск в 11:00: {next_run.strftime(\"%d.%m.%Y %H:%M\")}')
print(f'До запуска: {int((next_run - now).total_seconds())} секунд')
"
```

### 3. Запуск бота в тестовом режиме
```bash
# Создайте .env файл с тестовыми данными
echo 'TELEGRAM_BOT_TOKEN=test_token' > .env
echo 'ADMIN_CHAT_ID=123456789' >> .env
echo 'TZ=Europe/Moscow' >> .env

# Запустите бота
python3 bot.py
```

## Ожидаемое поведение

### При запуске:
1. Бот инициализируется и логирует информацию о таймзоне
2. Создается задача `lte_daily_check` с правильным временем первого запуска
3. Создается задача `scheduler_health_check` для мониторинга здоровья
4. Отправляется начальный статус всех роутеров

### В 11:00 ежедневно:
1. Выполняется проверка всех LTE модемов
2. Отправляется отчет с результатами проверки
3. Отмечается, что проверка выполнена сегодня
4. Планируется следующая проверка на завтра

### Мониторинг здоровья (каждые 6 часов):
1. Проверяется наличие задачи `lte_daily_check`
2. Если задача утеряна - автоматически пересоздается
3. Если сегодня не было проверки и уже после 11:00 - запускается принудительно

## Возможные проблемы и решения

### 1. Задача не создается
- Проверьте переменную окружения `TZ`
- Убедитесь, что `TELEGRAM_BOT_TOKEN` корректный
- Проверьте логи на ошибки

### 2. Проверка не выполняется в 11:00
- Используйте команду `/scheduler` для проверки статуса
- Используйте команду `/reschedule` для перезапуска
- Проверьте логи на ошибки выполнения

### 3. Дублирование проверок
- Проверьте, что в коде нет дублирующих задач
- Убедитесь, что используется только одна задача `lte_daily_check`
