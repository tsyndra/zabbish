# Настройка SSH для Keenetic роутеров

## Метод 1: SSH с паролем (для роутеров без поддержки SSH ключей)

### Настройка в Keenetic:
1. **Веб-интерфейс** → **"Управление"** → **"Общие настройки"**
2. Включите **"SSH-сервер"** (порт 22)
3. **"Управление"** → **"Пользователи"** 
4. Создайте/настройте пользователя admin с паролем
5. Дайте пользователю права администратора

### Конфигурация в routers.json:
```json
{
    "name": "40LET",
    "ip": "87.239.29.247",
    "description": "MAIN", 
    "ssh_access": {
        "username": "admin",
        "password": "ваш_пароль_ssh"
    },
    "lte_gateway": "192.168.0.1"
}
```

### Тестирование:
```bash
ssh admin@87.239.29.247
# Введите пароль
ping 192.168.0.1 -c 1
```

## Метод 2: SSH с ключами (для поддерживающих моделей)

### Если SSH ключи поддерживаются:
1. Генерация ключа:
```bash
ssh-keygen -t rsa -b 2048 -f keenetic_ssh_key
```

2. Добавление в Keenetic через веб-интерфейс или командную строку

3. Конфигурация:
```json
{
    "name": "40LET", 
    "ip": "87.239.29.247",
    "description": "MAIN",
    "ssh_access": {
        "username": "admin",
        "key_path": "/home/workdir/zabbiz-hati-tg-bot/keenetic_ssh_key"
    },
    "lte_gateway": "192.168.0.1"
}
```

## Метод 3: Альтернативы SSH

### SNMP мониторинг:
1. Включите SNMP в Keenetic
2. Используйте snmpget для проверки интерфейсов
3. Команда: `snmpget -v2c -c public IP_РОУТЕРА 1.3.6.1.2.1.2.2.1.8.X`

### HTTP API (некоторые модели):
1. Проверьте документацию модели
2. Используйте REST API endpoints

### Telnet (устаревший):
1. Включите Telnet в настройках
2. Подключение: `telnet IP_РОУТЕРА`

## Проверка LTE модема через SSH

Команды для проверки в Keenetic:
```bash
# Проверка интерфейсов
ip link show

# Проверка USB устройств  
lsusb | grep -i modem

# Проверка маршрутов
ip route | grep ppp

# Пинг LTE модема
ping 192.168.0.1 -c 1

# Статус 3G/4G соединения
cat /proc/net/dev | grep ppp
```

## Безопасность

1. **Измените стандартный порт SSH:**
   - В настройках укажите порт 2222 вместо 22
   - Обновите конфигурацию: `"ssh_port": 2222`

2. **Используйте сложные пароли**

3. **Ограничьте доступ по IP** (если возможно)

4. **Регулярно обновляйте прошивку роутера**
