# Router Monitor Bot

Telegram бот для мониторинга роутеров и LTE-модемов.

## Возможности

- Мониторинг доступности роутеров каждые 10 секунд
- Проверка статуса LTE-модемов через SSH
- Автоматические уведомления о недоступности/восстановлении
- Ежедневная проверка LTE-модемов в 11:00 по московскому времени
- Мониторинг здоровья планировщика

## Команды бота

- `/status` - показать текущий статус всех роутеров
- `/reschedule` - перезапустить планировщик ежедневных проверок
- `/test` - тестовая команда для проверки работы бота
- `/scheduler` - показать детальный статус планировщика

## Исправления планировщика (v2.0)

- Упрощена логика планировщика - убраны дублирующие задачи
- Улучшена обработка таймзон для корректной работы в 11:00
- Добавлен автоматический мониторинг здоровья планировщика
- Улучшена обработка ошибок с автоматическим восстановлением
- Добавлены команды для управления планировщиком

### 🔧 Новые функции:
- **Автоматическое восстановление** - если задача планировщика утеряна, она пересоздается
- **Мониторинг здоровья** - каждые 6 часов проверяется состояние планировщика
- **Предотвращение дублирования** - проверка выполняется только раз в день
- **Очистка старых записей** - записи старше 7 дней автоматически удаляются

## Установка и запуск

### Docker (рекомендуется)
```bash
docker-compose up -d
```

### Ручная установка
```bash
pip install -r requirements.txt
python3 bot.py
```

## Конфигурация

Создайте файл `.env` с переменными:
```
TELEGRAM_BOT_TOKEN=your_bot_token
ADMIN_CHAT_ID=your_chat_id
TZ=Europe/Moscow
MIN_OFFLINE_THRESHOLD=30
```

## Структура проекта

- `bot.py` - основной файл бота
- `routers.json` - конфигурация роутеров
- `check_lte_status.py` - скрипт для проверки LTE-модемов
- `Dockerfile` - конфигурация Docker образа
- `docker-compose.yml` - конфигурация Docker Compose

## Логи

Логи сохраняются в директории `logs/` с автоматическим ротацией.
