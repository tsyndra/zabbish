# 📱 Система мониторинга LTE-модемов

## 🎯 Описание

Система автоматического мониторинга LTE-модемов на роутерах с ежедневными проверками и отправкой отчетов в Telegram.

## ✨ Возможности

- ✅ **Автоматическая проверка LTE-модемов** каждый день в 11:00
- ✅ **Поддержка разных типов роутеров**:
  - MikroTik роутеры (через SSH)
  - Keenetic роутеры (через ping gateway)
- ✅ **Отправка отчетов в Telegram** с детальной статистикой
- ✅ **Логирование всех операций** для отладки
- ✅ **Cron-планировщик** для надежного выполнения

## 🚀 Быстрый старт

### 1. Настройка переменных окружения

Убедитесь, что в файле `.env` настроены:
```bash
TELEGRAM_BOT_TOKEN=ваш_токен_бота
ADMIN_CHAT_ID=ваш_chat_id
TZ=Europe/Moscow
```

### 2. Настройка cron задач

```bash
chmod +x setup_lte_cron.sh
./setup_lte_cron.sh
```

### 3. Тестирование

```bash
# Запуск проверки LTE-модемов
python test_lte_daily.py

# Отправка отчета в Telegram
python send_lte_report.py
```

## ⚙️ Конфигурация роутеров

### MikroTik роутеры (SSH)
```json
{
    "name": "ROUTER_NAME",
    "ip": "192.168.1.1",
    "description": "Описание",
    "ssh_access": {
        "username": "admin",
        "key_path": "./router_ssh_key"
    }
}
```

### Keenetic роутеры (LTE Gateway)
```json
{
    "name": "ROUTER_NAME",
    "ip": "192.168.1.1",
    "description": "Описание",
    "ssh_access": {
        "username": "admin",
        "password": "password"
    },
    "lte_gateway": "192.168.0.1"
}
```

## 📊 Формат отчетов

Отчеты отправляются в Telegram и содержат:
- 📱 Статус каждого LTE-модема
- 📊 Общую статистику доступности
- ⏰ Время последней проверки
- ❌ Детали ошибок для недоступных модемов

## 🐛 Устранение неполадок

### Проблема: SSH ключи не найдены
**Решение**: Убедитесь, что в `routers.json` указан правильный путь к SSH ключу:
```json
"key_path": "./router_ssh_key"
```

### Проблема: LTE gateway недоступен
**Решение**: Проверьте правильность IP адреса LTE gateway в конфигурации

### Проблема: Cron задачи не выполняются
**Решение**: Проверьте права доступа и перезапустите cron:
```bash
sudo service cron restart
```

### Проблема: Отчеты не отправляются в Telegram
**Решение**: Проверьте настройки бота в `.env` файле
